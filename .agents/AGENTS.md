# Luke-doku: Agent Rules & Project Guidelines

This file outlines the project structure, design patterns, and specific peculiarities of the **Luke-doku** repository to assist AI agents working on this codebase.

---

## 1. Project Overview & Tech Stack

Luke-doku is a Sudoku solver, generator, and analysis engine containing:
- **Rust Library (`crate/`)**: The core algorithm engine, puzzle generator, logical trail search solver, and WebAssembly bindings.
- **Web App (`www/`)**: A client-side TypeScript frontend built using Lit-HTML and Web Workers.

---

## 2. Key Project Quirks & Gotchas

Be extremely mindful of the following features and design constraints during development, as they differ from standard Sudoku applications:

### No Automated Pencil Marks
* Luke-doku **does not** automatically maintain or auto-trim pencil marks on the board.
* Users can place pencil marks for their own edification, but the board will never automatically generate or clean up candidates for them.

### Multiple Puzzle Solutions (10% of Puzzles)
* Puzzles generated by Luke-doku are not guaranteed to have a single unique solution.
* Approximately **10% of generated puzzles have 2 or 3 valid solutions**.
* **CRITICAL UI REQUIREMENT:** When completing a puzzle, you must explicitly state or display the **total number of solutions** in the final completion message/display.

### The "Trail" Mechanic (Speculative Elimination)
* Players can guess a candidate value placement in a cell to start a speculative "trail".
* If propagation of that guess leads to a logical error/contradiction, the player can disprove the candidate and eliminate it as a possibility.

### Game Review & Logical Trail Replay
* Players can review completed games and explore alternative routes to see where they missed deductions.
* The review page uses a **Web Worker slice-based architecture** to asynchronously search for logical disproofs.

## 3. Disproof Logic Architecture (v2)

When working on solver logic or the review page, refer to the documentation in [docs/](file:///Users/luke/code/luke-doku/docs/):
* **Current Design:** [nested_disproofs_design.md](file:///Users/luke/code/luke-doku/docs/nested_disproofs_design.md) details the **Productivity-First & Nested Disproof Search v2** used to handle Expert/Lunatic puzzles.
  * *Productivity-First:* Identifies erroneous candidates by comparing to solution grids, calculates elimination productivity (how many cells get solved downstream) via bitwise `Ledger` propagation, and runs sequential searches from highest to lowest productivity.
  * *Nested Disproofs:* Solve Lunatic puzzles recursively within speculative states rather than tracking complex multi-antecedent combinations.
* **Historical Reference:** [review_page_disproofs_design.md](file:///Users/luke/code/luke-doku/docs/review_page_disproofs_design.md) documents the older combination-based search approach (superseded by v2).
